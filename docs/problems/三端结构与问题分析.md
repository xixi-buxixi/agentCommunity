# 项目三端结构与问题分析

## 1. 前端（pulse-frontend）

### 结构与逻辑
- Vue 3 + Pinia + Router，组件划分合理，视觉风格统一，API 封装良好。

### 存在问题
1. Pinia store 在 Axios 拦截器中多次实例化，可能导致响应式失效。
2. API 版本切换方式不统一，维护易混乱。
3. 部分组件 props 校验不严格，易引发渲染异常。
4. 前端表单缺少输入校验，存在安全隐患。
5. 进度条/状态边界未做防御，极端情况可能异常。

---

## 2. Java 后端（pulse-backend）

### 结构与逻辑
- 标准 Spring Boot 分层结构，数据库设计健全，接口实现规范，调度任务与权限控制合理。

### 存在问题
1. Agent 死亡判定和 token 扣减存在并发 race condition，可能导致重复死亡或超卖。
2. 软删除未全局过滤，可能导致脏数据。
3. 行为幂等性和唯一约束部分接口未严格校验。
4. API Key 解密逻辑缺乏权限隔离，有泄漏风险。
5. JWT 校验需确保所有敏感接口均加保护。

---

## 3. Python 侧（pulse-ai-side）

### 结构与逻辑
- FastAPI，服务拆分合理，Prompt 构建与 LLM 封装健全，异常兜底完善。

### 存在问题
1. 仅支持 OpenAI 兼容 API，无法直接适配 Claude/文心一言等异构协议。
2. Prompt 注入防护仅靠正则，易被变体绕过。
3. 所有接口无鉴权、无频控，生产环境安全性不足。
4. LLM 异常均返回 200，action=ignore，不利于问题定位。
5. 上下文仅做长度截断，未做语义筛选，token 利用率不高。
6. 解析失败未记录原始 LLM 输出，debug 不便。
