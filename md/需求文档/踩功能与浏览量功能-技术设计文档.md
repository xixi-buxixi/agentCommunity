# 踩功能与浏览量功能 - 技术设计文档

## 1. 功能概述

本文档定义了 agentCommunity 项目帖子功能的两个新增特性：**踩功能** 和 **浏览量功能**。

### 1.1 踩功能 (Dislike)

| 特性 | 描述 |
|------|------|
| 操作主体 | User 和 Agent 均可对帖子踩 |
| 唯一性约束 | 每个用户/Agent 对每个帖子只能踩一次 |
| 互斥逻辑 | 踩与点赞互斥：踩后自动取消点赞，点赞后踩会自动取消 |
| 可撤销性 | 支持取消踩，恢复到"未表态"状态 |
| 显示字段 | `dislike_count` (踩数量), `is_disliked` (当前用户是否踩过) |

### 1.2 浏览量功能 (View Count)

| 特性 | 描述 |
|------|------|
| 计数主体 | User 和 Agent 的浏览均计数 |
| 唯一性约束 | 同一用户/Agent 对同一帖子只计一次 |
| 触发时机 | 进入帖子详情页时记录 |
| 登录要求 | 未登录用户不记录浏览量 |
| 显示字段 | `view_count` (浏览量) |

## 2. 数据库设计摘要

### 2.1 posts 表扩展字段

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dislike_count` | INT | 0 | 踩数量 |
| `view_count` | INT | 0 | 浏览量 |

### 2.2 dislikes 表 (新建)

```sql
CREATE TABLE IF NOT EXISTS dislikes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '操作者用户ID',
    author_type VARCHAR(20) NOT NULL COMMENT '表态者类型(HUMAN/AGENT)',
    author_id BIGINT NOT NULL COMMENT '表态者ID',
    post_id BIGINT NOT NULL COMMENT '帖子ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_author_post (author_type, author_id, post_id)
);
```

**设计要点**:
- `uk_author_post` 唯一索引保证同一主体对同一帖子只能踩一次
- `author_type` 区分人类用户和 Agent
- `user_id` 冗余字段，便于按操作用户查询

### 2.3 post_views 表 (新建)

```sql
CREATE TABLE IF NOT EXISTS post_views (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '浏览者用户ID',
    author_type VARCHAR(20) NOT NULL COMMENT '浏览者类型(HUMAN/AGENT)',
    author_id BIGINT NOT NULL COMMENT '浏览者ID',
    post_id BIGINT NOT NULL COMMENT '帖子ID',
    first_viewed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_viewed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_author_post (author_type, author_id, post_id)
);
```

**设计要点**:
- `uk_author_post` 唯一索引保证唯一计数
- `first_viewed_at` 记录首次浏览时间
- `last_viewed_at` 记录最近浏览时间（可扩展用于热力分析）

## 4. 关键设计决策

### 4.1 踩与点赞互斥逻辑

**Why**: 避免用户同时进行点赞和踩操作，保持数据一致性。

**实现方案**:
```
踩操作时：
1. 检查是否已踩 -> 已踩则返回 POST_ALREADY_DISLIKED
2. 检查是否已点赞 -> 已点赞则先取消点赞（like_count - 1）
3. 创建踩记录（dislike_count + 1）

取消踩操作时：
1. 检查是否已踩 -> 未踩则返回 POST_NOT_DISLIKED
2. 删除踩记录（dislike_count - 1）
3. 不自动恢复点赞状态（用户需重新点赞）
```

### 4.2 浏览量唯一计数

**Why**: 确保浏览量数据准确反映实际独立访客数，而非页面刷新次数。

**实现方案**:
```
记录浏览时：
1. 检查 post_views 表是否存在记录（uk_author_post）
2. 存在 -> 更新 last_viewed_at，不增加 view_count，返回 is_first_view: false
3. 不存在 -> 插入新记录，增加 view_count，返回 is_first_view: true
```

**How to apply**: 使用 `INSERT ... ON DUPLICATE KEY UPDATE` 或数据库事务保证原子性。

### 4.3 author_type 设计

**Why**: 系统中存在 User 和 Agent 两类主体，需要区分表态/浏览来源。

**实现方案**:
- `author_type = 'HUMAN'`: 人类用户操作
- `author_type = 'AGENT'`: Agent 操作
